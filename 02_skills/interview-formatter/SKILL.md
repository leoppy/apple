# 面试评价整理工具

将用户输入的笼统面试评价内容，整理成规范的结构化格式。

## 使用场景

- 面试官在面试过程中或面试后，需要快速记录候选人的表现
- 将口语化、碎片化的评价内容转换为标准化的面试评价报告
- 自动计算面试结果和通过状态

## 输出格式规范

整理后的面试评价必须包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 工作经验 | 候选人的工作年限或学历背景 | 2年 / 硕士应届 / 3年Java开发 |
| 沟通能力 | 候选人的表达、沟通、协作能力评价 | 表达清晰，逻辑性强 |
| 理解能力 | 候选人对问题的理解和把握能力 | 能快速理解需求，抓住重点 |
| 应变能力 | 候选人面对问题的应变和解决能力 | 遇到难题能灵活调整思路 |
| 专业能力 | 候选人的技术水平、专业知识掌握情况 | 算法基础扎实，项目经验丰富 |
| 职级建议 | 建议的职级定位 | T2 / T3 / P6 |
| 面试结果 | 基于工作经验/职级的打分（满分10分，6分及格） | 7/10 |
| 是否通过 | 通过或不通过（≥6分为通过） | 通过 / 不通过 |

## 评分规则

- **满分**：10分
- **及格线**：6分
- **通过标准**：面试结果 ≥ 6分
- **评分依据**：基于候选人的工作经验、职级定位以及各项能力的综合表现

## 处理流程

### 1. 接收用户输入

用户可能以各种形式输入面试评价，例如：
- 口语化描述："这个候选人硕士刚毕业，表达还不错，技术基础可以，给个T2吧，我觉得7分"
- 碎片化信息："2年经验，沟通OK，理解能力一般，专业能力还行，建议T2"
- 混合格式："工作2年，沟通能力强，其他都还可以，T2，7分"

### 2. 信息提取与补全

从用户输入中提取以下信息：
- **工作经验**：识别年限、学历、岗位等关键词
- **各项能力评价**：提取对沟通、理解、应变、专业能力的描述
- **职级建议**：识别职级标识（T2、T3、P6等）
- **面试结果**：提取分数或根据描述推断分数

**如果某些字段缺失**：
- 如果用户没有明确提到某项能力，输出时该字段留空或填写"未评价"
- 如果没有明确分数，根据整体评价推断一个合理分数（参考：优秀8-10分，良好7分，一般6分，较差5分以下）

### 3. 自动判断通过状态

根据面试结果自动判断：
- 面试结果 ≥ 6分 → 是否通过：**通过**
- 面试结果 < 6分 → 是否通过：**不通过**

**注意**：判断逻辑不需要在输出中解释，直接给出结果即可。

### 4. 格式化输出

使用表格形式输出，清晰美观：

```
| 评价项 | 内容 |
|--------|------|
| 工作经验 | 2年 |
| 沟通能力 | 表达清晰，逻辑性强 |
| 理解能力 | 能快速理解需求 |
| 应变能力 | 思路灵活 |
| 专业能力 | 技术基础扎实 |
| 职级建议 | T2 |
| 面试结果 | 7/10 |
| 是否通过 | 通过 |
```

## 输出要点

1. **简洁明了**：每项评价控制在1-2句话，不超过30字
2. **客观中立**：基于事实描述，避免主观情绪化表达
3. **结构统一**：所有面试评价使用相同的表格格式
4. **自动计算**：通过状态由系统自动判断，无需用户手动填写
5. **保留原意**：尽量保留用户原始评价的核心意思，只做格式化整理

## 示例

### 示例 1：完整信息

**用户输入**：
```
这个候选人硕士应届，表达很清楚，理解能力不错，遇到问题能想办法，专业基础扎实，建议给T2，我打7分
```

**输出**：
```
| 评价项 | 内容 |
|--------|------|
| 工作经验 | 硕士应届 |
| 沟通能力 | 表达清楚 |
| 理解能力 | 理解能力不错 |
| 应变能力 | 遇到问题能想办法 |
| 专业能力 | 专业基础扎实 |
| 职级建议 | T2 |
| 面试结果 | 7/10 |
| 是否通过 | 通过 |
```

### 示例 2：信息不完整

**用户输入**：
```
2年Java开发，沟通还行，技术一般，给T2，6分吧
```

**输出**：
```
| 评价项 | 内容 |
|--------|------|
| 工作经验 | 2年Java开发 |
| 沟通能力 | 沟通还行 |
| 理解能力 | 未评价 |
| 应变能力 | 未评价 |
| 专业能力 | 技术一般 |
| 职级建议 | T2 |
| 面试结果 | 6/10 |
| 是否通过 | 通过 |
```

### 示例 3：不通过案例

**用户输入**：
```
工作1年，表达不太清楚，理解有点慢，专业能力偏弱，建议再看看，5分
```

**输出**：
```
| 评价项 | 内容 |
|--------|------|
| 工作经验 | 1年 |
| 沟通能力 | 表达不太清楚 |
| 理解能力 | 理解有点慢 |
| 应变能力 | 未评价 |
| 专业能力 | 专业能力偏弱 |
| 职级建议 | 待定 |
| 面试结果 | 5/10 |
| 是否通过 | 不通过 |
```

## 注意事项

1. **保护隐私**：不在输出中包含候选人姓名、联系方式等敏感信息
2. **客观公正**：基于用户提供的信息进行整理，不添加主观判断
3. **灵活适配**：能够处理各种口语化、非标准化的输入
4. **快速响应**：整理过程应快速完成，不需要额外确认
5. **一致性**：同一场面试的多次整理应保持格式一致

## 扩展功能（可选）

如果用户需要，可以提供以下扩展功能：
- 批量整理多个候选人的面试评价
- 导出为 Excel 或 CSV 格式
- 生成面试评价统计报告
- 对比不同候选人的评价结果
