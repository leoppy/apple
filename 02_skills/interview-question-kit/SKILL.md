# 面试出题 Skill

## 目标
帮助面试官从通用题库高效出题：
1. 通用题准备：从题库按标签筛题。
2. 简历适配：结合简历画像从通用题库中挑选更匹配的题。

## 目录约定
- `question_bank/common/`：通用题库（每题一个 md）。
- `outputs/`：每次产出的面试题单，文件名必须包含人名。
- `scripts/`：辅助脚本。
- `references/`：标签与模板说明。

## 单题文件格式（强约束）
每个题目必须是独立 `.md`，并包含如下 frontmatter 与内容段落：

```md
---
id: G001
tags: [通用基础, 沟通表达]
type: 行为题
difficulty: 校招
---

# 题目
...

# 参考答案
...

# 追问
...
```

## 工作流
1. 明确出题目标：岗位、级别、题量、标签偏好。
2. 如果用户给了简历路径（尤其 PDF），先提取简历关键信息：项目、技术栈、职责、亮点/风险点。
3. 从 `question_bank/common/` 按标签筛选题目并组合题单。
4. 输出到 `outputs/<人名>_面试题.md`。
5. 输出内容包含：题目、考察点、参考答案、追问。

## 标签筛选原则
- 优先覆盖 3 类：`通用基础`、`项目深挖`、`行为题`。
- 若候选人有明确技术栈，再加 `工程习惯` 或对应技术标签。
- 避免全是八股；至少 1 题与候选人项目直接绑定。

## 产出模板

```md
# <姓名> 面试题（<岗位/场景>）

## 候选人摘要
- ...

## 面试题
### Q1（标签：项目深挖, 场景题）
- 题目：...
- 考察点：...
- 参考答案：...
- 追问：...
```

## 可选脚本
- 提取简历文本：
  - `python scripts/extract_resume_text.py --pdf "<简历路径>" --out "outputs/<人名>_resume.txt"`
- 按标签生成题单：
  - `python scripts/build_interview_set.py --candidate "<姓名>" --tags "项目深挖,行为题" --count 5`
