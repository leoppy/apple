# 题目
请你基于 `Python + Requests + Pytest + YAML + Allure` 设计一套接口自动化框架，并现场讲清楚：
1) 目录分层怎么设计（数据、用例、公共方法、配置）？
2) 如何处理鉴权 token、环境切换、测试数据隔离？
3) 如何避免“脚本能跑但结果不可信”？

# 参考答案
关键点应包括：
- 分层清晰：`tests/`、`data/`、`common/`、`config/`、`reports/`，业务与工具解耦。
- 配置管理：测试/预发配置分离，敏感信息走环境变量，不硬编码。
- 数据治理：唯一测试数据、前置清理、可重复执行、失败可复盘。
- 结果可信度：断言业务语义而非仅状态码；失败日志可追踪请求/响应；Allure 报告包含步骤与附件。

# 追问
如果 CI 中自动化用例经常“偶发失败”，你会如何区分是环境波动还是脚本质量问题？
