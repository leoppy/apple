# 题目
假设你负责某内容平台的“购买+解锁+阅读”核心链路测试，请设计测试方案：
1) 功能与接口测试覆盖哪些关键场景？
2) JMeter 压测你会如何设定并发模型与核心指标？
3) 自动化回归中哪些用例必须进入冒烟集？

# 参考答案
一个合格答案应包含：
- 功能链路：余额不足、重复购买、幂等、并发扣费、异常回滚、权限校验、缓存一致性。
- 接口层：状态码、返回字段、金额精度、订单状态流转、日志 traceId 关联。
- 压测设计：按读多写少业务模型设定阶梯并发，关注 RT(P95/P99)、吞吐、错误率、数据库连接与锁等待。
- 自动化策略：把登录、下单/支付、解锁校验、权限校验、回滚校验放入冒烟；高频回归走 nightly。

# 追问
如果压测发现 P99 延迟飙升但平均延迟正常，你怎么继续定位？
