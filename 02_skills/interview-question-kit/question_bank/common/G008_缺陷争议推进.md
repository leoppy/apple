# 题目
假设在一个跨部门项目中，开发负责人认为某个缺陷“不是 bug，不修也能上线”，但你判断有线上风险。你会如何推进？

# 参考答案
优秀回答通常会：
- 用影响面和复现证据说话（影响用户范围、触发条件、业务损失）。
- 给出分级和备选方案（必须修复/降级上线/加监控与回滚预案）。
- 拉齐产品、开发、测试三方决策，形成书面结论与责任归属。
- 保证上线后可观测（告警阈值、回归清单、值班预案）。

# 追问
如果最终决定“带风险上线”，你会要求补哪些最小防护措施？
